## Mission (unchanged intent)

Build one squeeze BOOM algo. Run it through three optimisation “councils” (conceptually
27 agents). Gate winners against thresholds. **Recurse with documented widening only
after OOS/holdout is defined** — see ALGO_DEV_MACHINE_OPT.md. Fake metrics = removal
from council (re-run).

## Phase 0 — master signal (target: `BoomMasterAlgo`)

**Today:** canonical entries already live in `ds_app/boom_backtest.py` — `_boom_features`
(BB/KC squeeze, release, RVOL, trend vote, Darvas or arrows path, ATR gate, first-half).

**Target module:** consolidate a single boolean entry `pd.Series` API (e.g. in
`boom_vectorbt.py` or a thin `boom_master_signal.py`) that **only** calls shared
boom_backtest helpers — no duplicate squeeze math.

It must:

1. Use existing squeeze = BB inside Keltner (same definitions as `_boom_features`).
2. Compute `boom_vote` (+ squeeze, release, RVOL, trend) and `boom_score` blend.
3. Apply ATR gate and first-half filter (09:30–<12:45 ET intraday; daily bars exempt).
4. Emit one boolean entry series per (symbol, param set). Councils tune **parameters**,
   not divergent signal definitions.

## Phase 4 — winner gate (example thresholds)

Treat as **tunable** — hard numeric gates on small samples create false negatives/positives.

```text
sharpe > 1.0              # may fail on 60d 5m even for decent edges
max_dd_pct < 15.0
trades >= 5
win_rate_pct > 40.0
pca_variance_explained > 0.40   # if PCA used
live_sharpe > 0.5              # define “live” window explicitly
symbols_with_entries >= 3
```

If no winner: **widen grid one step** *and* **declare new OOS segment** — do not only
widen on the same path (see ALGO_DEV_MACHINE_OPT **G5**).

## Recursive opt loop (implementation sketch)

```python
iteration = 0
max_iter = 20   # safety cap — “never stop” in prod = stop + widen + document, not infinite CPU
while not winners and iteration < max_iter:
    iteration += 1
    c1 = run_council_1(params)
    c2 = run_council_2(c1.survivors)
    c3 = run_council_3(c2.survivors)
    winners = gate(c3)
    if not winners:
        params = widen(params)
        log(f"iter {iteration}: no winner — widened params, see OOS note")

if not winners:
    log("no winner after max_iter — publish diagnostics, do not fake metrics")
```

## Builder rules (non‑negotiable)

1. Iteration **cap** 20 (or configurable) — escalate to human if exhausted.
2. Every agent logs **top‑3 + metrics + repro URL** (`/boom-visual/...`, `/vectorbt-expansion/?format=json&...`).
3. PCA on **real** numeric matrix — no invented eigenvalues.
4. Live validation: **real yfinance** in dev; **synthetic** only in CI (existing tests patch loaders).
5. Eliminated agents log **metric + threshold**.
6. Hallucinated metrics → **remove** from council table; re-run.
7. **Future:** `boom_vectorbt_winners.json` (or manifest in `format=json` export) — not written by repo today; add with tests when implemented.
8. Dashboard / “run cycle” UI described in original spec = **not in this repo** — treat as product backlog or MISSION embed.
